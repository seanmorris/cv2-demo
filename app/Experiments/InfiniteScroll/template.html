experimental scrollers

These scroll boxes automatically remove any DOM elements that are no longer visible to the user. This allows us to make large scrollable boxes without wasting large amounts of computing power on DOM calculations for records that may never become visible.

simple scroller

This scroller uses an array of [[simpleRows|thousands]] integers.

Building large arrays can have significant overhead.

[[arrayScroller]]

string scroller

This scroller uses a RecordSet that returns 1,000,000 strings.

This list was able to populate quickly because RecordSets defer loading.

[[stringScroller]]

view scroller

This scroller uses a RecordSet that returns 1,000,000 curvature View objects.

There is a 250ms simulated load time on each row.

[[viewScroller]]

table scroller

This scroller uses flexbox, and a custom template to simulate a table. Its RecordSet returns [[rows|thousands]] plain objects.

The number of records and row height can be altered on the fly as well, this is common to all Scrollers.

Counting both the texboxes & sliders, there are 2,000,002 input fields avaiable below, each mapped to an object in memory.

NOTE: This is currently limited by the scroll-height limit of 33,554,400 pixels imposed by webkit.
