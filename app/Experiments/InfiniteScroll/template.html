Experimental Scrollers

These scroll boxes automatically remove any DOM elements that are no longer visible to the user. This allows us to make large scrollable boxes without wasting large amounts of computing power on DOM calculations for records that may never become visible.

Simple Scroller

This scroller uses an array of [[simpleRows|thousands]] integers.

Arrays and RecordSets are interchangeable here.

[[arrayScroller]]

String Scroller

This scroller uses a RecordSet that returns strings.

An array of strings could easily have been used instead of a RecordSet.

[[stringScroller]]

View Scroller

This scroller uses a RecordSet that returns curvature View objects.

There is a 250ms simulated load time when the view is first accessed.

[[viewScroller]]

Grid Scroller

This scroller uses CSS grid, and a custom template to simulate a table. Its RecordSet returns plain objects.

The number ([[rows|thousands]]) of records and row height can be altered on the fly as well, this is common to all Scrollers.

Counting both the texboxes & sliders, there are 2,000,002 input fields avaiable below, each mapped to an object in memory.

NOTE: This is currently limited by the scroll-height limit of 33,554,400 pixels imposed by webkit.

[[gridScroller]]
Position: [[gridScroller.args.scrollTop]] / [[gridScroller.args.scrollMax]]
Speed: [[gridScroller.args.speed]] px/s
