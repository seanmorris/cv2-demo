theme

Themes allow you to map arbitrary types to Views, and if necessary, views to their templates for an added level of abstraction. This behavior can be supplemented with additional views and templates at runtime.

Themes can fallback on other themes if they don't have the View or Template for the given object.

Usage:

All the editors on this page are linked, so modifying any files and clicking execute will propagate to the results tab of the first one.

Types

These are the objects that will be mapped to views. They will be given Bindable wrappers, then provided to Views as their args property.

Note that you'll need to modify the object through the Bindable wrapper to propagate the changes up to the view.

TypeX.js achieves this by creating the Bindable in the constructor. Subsequent calls to Bindable.make will return the same reference. TypeZ.js returns the bindable from the constructor, so other objects will have the Bindable reference by default. Its child class, TypeZAlpha then does not need to repeat this pattern. the value in TypeY is constant, so we do not make it Bindable by default.
