bindables

Bindable.make() & [Object Bindable].bindTo()

Objects can be made into bindables with the Bindable.make({...}) funciton. Just pass almost any object as the only parameter and you'll get back a proxied object. Call .bindTo() on the object to watch for changes to its values.

	class Foobar {};

	const foo = Bindable.make( new Foobar );

	foo.bindTo('x', v => console.log(v));

	foo.x = 5;

watching all propteries

Passing a callback as the first parameter without a property name will watch all properties on the object. The second parameter to the callback is the name of the property being set.

	class Foobar {};

	const foo = Bindable.make( new Foobar );

	foo.bindTo((v,k) => console.log(k, v));

	foo.x = 5;
	foo.y = 6;
	foo.z = 7;

callback(v,k,t,d,p)

The function supplied to bindTo will be called with the following parameters when the value changes. 
	value    - the new value being set
	key      - the property being set
	target   - the target object
	delete   - true if the property is being deleted, false otherwise.
	previous - the previous value of the property

bindTo options

The last parameter to bindTo() is an options object.

The following keys may be supplied to alter the way the callback is called.

now

Pass {now: false} in your options to prevent your callback from being called on the initial bind.

By default, the callback will be run once when the bind operation happens.

delay

Delay before callback, in milliseconds. Note This is NOT a throttle, this will simply offset the callback's execution to a later time.

throttle

Prevent the callback from running more than once every X milliseconds.

wait

Wait for an oft-changing value to "settle" before running the callback. If the changes, then stops changing for at least X milliseconds, the callback will run once.

idle

Rather than executing the callback immediately, defer to requestIdleCallback(). Falls back to requestAnimationFrame() if requestIdleCallback is unavailable.

frame

Rather than executing the callback immediately, defer to requestAnimationFrame().

removeWith

Rather than executing the callback immediately, defer to requestAnimationFrame().

children

Run the callback both when the property is written or when ANY the child object's proprties change. 5 parameters will be supplied to the callback in addition to the original v,k,t,d,p to represent changes to the child object.
